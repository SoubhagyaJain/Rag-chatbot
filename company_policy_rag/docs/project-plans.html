A structured record of every major engineering plan — policy relevancy recovery (Track B), dual-corpus guidebook extension (Track A), code validation, corpus scoping, and enumeration tuning. Source of truth for current status: README3.md.

Engineering Timeline

Fourteen acts spanning baseline setup through guidebook quality tuning. Acts 1–9 are policy-corpus recovery; Acts 10–14 are Track A multi-corpus work (2026-06-18).

Act 1 — Baseline & Reranker

Indexed handbook PDF, added ChromaDB + bge-reranker. Context Precision ~0.53, Relevancy ~0.71.

Act 2 — Strict Grounding Experiment

Faithfulness hit 1.00 but Relevancy collapsed to 0.42. Balanced mode chosen as default.

Act 3 — The Regression (run 091001)

Relevancy fell to 0.40 — worst score. Guard nuking answers, prompt bloat, vocabulary gaps.

Act 4 — Relevancy Recovery (Phases 1–4)

0.40 → 0.747 via guard fix, prompt trim, query augmentation, normalize_balanced_answer().

Act 5 — Streamlit UI

Replaced Chainlit. Migrated ReActAgent to LlamaIndex 0.14 workflow API.

Act 6 — Citation Trust Fix

Stopped parallel retriever in UI. SourceTrackingQueryEngine + cited-only display.

Act 7 — Mandatory [Source N] Prompts

Hard citation rules in balanced prompt + agent system prompt. Example J (BAD) added.

Act 8 — Chroma Telemetry & Index Health

NoOpProductTelemetry, probe_chroma_index(), cache reset on Streamlit startup.

Act 9 — Docker Deployment

Streamlit container + host Ollama via host.docker.internal. Build verified on Windows.

Act 10 — Guidebook Corpus + Golden v2

AI Agents guidebook indexed (228 chunks). Combined golden set expanded to 60 cases (25 policy + 35 guidebook). Caption inheritance fixes section_path metadata.

Act 11 — Hybrid BM25 + Parent Retrieval

Track A Phase 2: dense + BM25 RRF fusion, parent-child chunking with parent expand at generation time.

Act 12 — Code Validation Pipeline

Track A Phase 3: post-generation code validation, self-correct once, strip_code fail mode. Weak subset 143246: pass 100%, fallback 0%.

Act 13 — Corpus Scope + Re-index

ScopeFilteredRetriever post-filters by source_file. Force re-index: guidebook section_path unknown 49% → 0%.

Act 14 — Enumeration Tuning

Comprehensive list retrieval, guidebook term expansion, section-diverse rerank, enum few-shots. Subset 160052: enum rel 0.84. Full 35-case 164848: rel 0.700 — gate passed.

Plan 1 — Answer Relevancy Recovery (Phases 1–4)

Near Target
Goal
Relevancy ≥ 0.75, Faithfulness ≥ 0.90
Trigger
Regression run 091001 — Relevancy 0.40
Outcome
0.747 relevancy (+87% from floor)

Problem

Balanced faithfulness guard replaced entire answers with abstention on UNSUPPORTED. Prompt bloat taught abstention prior. Vocabulary mismatch and double-ending patterns killed per-case scores.

Approach

  1. Phase 1: Guard keeps answer on reject; strip double-ending abstention suffixes (src/generation.py)
  2. Phase 2: Trim few-shots; add semantic mapping; augment_query_with_policy_terms()
  3. Phase 3: RERANKER_TOP_N=6; context-aware relevancy judge
  4. Phase 4: normalize_balanced_answer(); at-will/disciplinary/outside-employment prompts

Plan 5 — Chroma Telemetry & Index Health

Completed
Goal
Silence telemetry errors; fix false "no index" in Streamlit
Trigger
posthog signature mismatch; Chunks: 0 while 81 existed
Outcome
Clean logs; real chunk count in diagnostics

Problem

capture() takes 1 positional argument but 3 were given on every Chroma start. SharedSystemClient cached stale settings causing silent index failure.

Approach

  1. NoOpProductTelemetry — never calls posthog (src/chroma_telemetry.py)
  2. Pin posthog>=2.4.0,<3.0.0 in requirements
  3. probe_chroma_index() — always reports real chunk count + error
  4. reset_chroma_client_cache() on Streamlit startup + retry button

Files Touched

  • src/chroma_telemetry.py (new)
  • src/indexing.py
  • app/streamlit_app.py
  • tests/test_chroma_telemetry.py
